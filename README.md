# 🗞 News Digest Bot

Двічі на день (ранок + вечір) збирає новини з RSS, робить стисле саммарі через LLM
(**Claude → Gemini → Groq**, фолбек) і надсилає дайджест у Telegram.
Працює на **GitHub Actions** — власний сервер не потрібен.

Теми за замовчуванням: **Україна + геополітика** та **Технології / AI** (редагується в `config.py`).

## Структура

```
config.py            # RSS-джерела + налаштування
fetcher.py           # збір і фільтрація новин з RSS
store.py             # SQLite-пам'ять: дедуплікація між випусками
summarizer.py        # саммарі через LLM з фолбеком
telegram_client.py   # відправка в Telegram
main.py              # точка входу
requirements.txt
.github/workflows/news-digest.yml
```

## Швидкий старт

### 1. Створити бота і дізнатись chat_id
1. У Telegram напиши [@BotFather](https://t.me/BotFather) → `/newbot` → отримаєш **TELEGRAM_BOT_TOKEN**.
2. Напиши щось своєму боту, потім відкрий
   `https://api.telegram.org/bot<ТОКЕН>/getUpdates` — у полі `chat.id` буде **TELEGRAM_CHAT_ID**.

### 2. Ключі LLM
Достатньо **хоча б одного** (решта — резерв):
- `ANTHROPIC_API_KEY` (Claude),
- `GEMINI_API_KEY` (Google AI Studio),
- `GROQ_API_KEY` (Groq).

### 3. Розгортання на GitHub Actions
1. Заливаєш репозиторій на GitHub.
2. **Settings → Secrets and variables → Actions → New repository secret** — додаєш:
   `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`, і потрібні `*_API_KEY`.
3. Готово. Workflow сам запуститься за розкладом. Перевірити вручну:
   **Actions → News Digest → Run workflow** (можна задати `slot`).

### Локальний запуск
```bash
pip install -r requirements.txt
cp .env.example .env      # заповни ключі
export $(grep -v '^#' .env | xargs)
python main.py --slot morning
```

## Розклад і час

Cron у GitHub Actions **завжди в UTC** і не враховує переходу на літній час.
Зараз (літо, Київ = UTC+3): `05:00 UTC → 08:00`, `17:00 UTC → 20:00`.
Взимку ті самі рядки дадуть 07:00 / 19:00 — за потреби зсунь на `0 6` та `0 18`
у `.github/workflows/news-digest.yml`. Сам слот (ранок/вечір) визначається
автоматично за київською годиною.

## Налаштування

| Що | Де |
|---|---|
| Додати/прибрати джерела | `FEEDS` у `config.py` |
| Теми/секції | ключі `FEEDS` + `CATEGORY_EMOJI` |
| Вікно свіжості | `LOOKBACK_HOURS` (13 год) |
| Стиль дайджесту | `SYSTEM_PROMPT` та правила у `summarizer.py` |
| Моделі | `CLAUDE_MODEL` / `GEMINI_MODEL` / `GROQ_MODEL` |

## Дедуплікація між випусками

Щоб та сама новина не потрапляла і в ранковий, і у вечірній випуск, надіслані
новини запам'ятовуються у SQLite (`store.py`, файл `state.db`). Дубль ловиться
за посиланням **або** за нормалізованим заголовком (та сама подія з іншого джерела).

- Новина позначається «переглянутою» **лише після успішної відправки** — тож при
  збої LLM чи Telegram нічого не губиться й піде наступного запуску.
- Історія тримається `RETENTION_DAYS` (3 дні), далі чиститься автоматично.
- На GitHub Actions `state.db` персиститься між запусками через `actions/cache`
  (rolling-ключ). Кеш може евіктнутись після кількох днів простою — але оскільки
  бот працює двічі на день, це не проблема; у найгіршому разі одна новина
  повториться.

> Альтернатива кешу — комітити `state.db` назад у репо. Кеш обрано як чистіший
> варіант (без комітів і без прав на запис у репозиторій).

## Стійкість
- Кожна стрічка парситься окремо: недоступне джерело просто пропускається.
- LLM-провайдери пробуються по черзі — якщо перший впав, працює наступний.
- Якщо Telegram відхилив HTML — повідомлення йде без розмітки.
- Якщо з минулого випуску нічого нового — бот нічого не шле (без спаму).

## Можливі апгрейди

- Фільтр за ключовими словами (напр. підсилити AI-тематику).
